# Backend Comparison

## Python

* Framework: FastAPI.
* Strengths: mature RAG ecosystem, quick iteration, strong library support.
* Project role: first complete implementation and reference behavior.

## Java

* Intended framework direction: Spring Boot with Spring AI concepts.
* Strengths: enterprise patterns, strong dependency injection, production operations.
* Project role: show how a structured Java backend hosts the same RAG modules.

## Go

* Framework direction: Go HTTP service with explicit RAG modules.
* Strengths: simple deployment, clear control flow, explicit infrastructure integration.
* Project role: show the lower-level mechanics of retrieval, embeddings, and memory.

## Comparison Criteria

* How much framework code is needed.
* How explicit each RAG stage remains.
* How storage integrations are modeled.
* How easy it is to test the pipeline.
* How naturally memory and query rewriting fit the ecosystem.
